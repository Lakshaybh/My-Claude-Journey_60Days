Architecture & extension

How this system is built

Agent responsibilities

Planner

Turns the brief into a structured outline once, before drafting begins.

Executor

Writes the first full draft from the outline.

Evaluator

Scores the current draft 0-100 against a fixed rubric, every round.

Critic

Names concrete weaknesses in the current draft, independent of the score.

Improver

Rewrites the draft using the prior evaluation and critique.

Memory manager

Condenses each round into one durable note carried into the next round's prompts.

Safety monitor

Flags factual overreach or unsafe claims in the current draft each round.

Final reviewer

Signs off once a stop condition fires, independent of the Evaluator's score.

Extension ideas

  • Swap the rubric for a domain-specific one (legal, medical, financial) and re-run the same loop unchanged.
  • Add a second Critic with an adversarial persona and require both to agree before the Improver runs.
  • Persist memory across sessions so recurring topics start from prior learnings.
  • Replace the single Evaluator score with a panel of three evaluators and average their scores.
  • Stream tokens instead of waiting for full responses, for a faster perceived loop.

Prompts to go further

  • "Add a human-in-the-loop approval gate before the Final Reviewer runs."
  • "Generalize this studio so the roster and rubric are chosen per workflow, not hardcoded."
  • "Add a cost tracker that estimates token spend per round."
