Assistant brief

Built for personal brand builders — solo professionals turning expertise into an audience. The single most important outcome per session: leave with content they can post today, plus the strategic "why" so they can replicate it themselves next time.

Assistant "brain" design

Brandline's behavior is still specified as a formal system prompt — the same document you'd hand to an LLM — structured in five parts: role (a senior content strategist persona, not a generic writer), scope (personal brand content only — it declines unrelated requests), constraints (platform-native formatting, no generic advice, no fabricated stats/claims about the user), output format (a fixed Markdown structure: Strategic Angle → Hooks → Ready-to-Post Draft → Why This Works → Next Steps), and edge-case handling (vague briefs, off-topic requests, harmful/manipulative content requests, and empty topics). Instead of sending this prompt to a hosted model, this build encodes the same rules directly as a deterministic JavaScript template engine — so the app needs no API key, no network call, and no cost to run.

UI decisions

  • Chip selectors over dropdowns for platform/goal/tone — faster to scan, and the active state doubles as a visual summary of the brief.
  • Sticky form panel so the brief stays visible while scrolling a long generated strategy.
  • Skeleton + pulse-orb loading state instead of a spinner — even though generation is instant, a brief simulated "thinking" beat sets the expectation that a structured document is being composed.
  • Light, editorial-grade theme (indigo → violet → teal accent gradient) to feel like a premium creative tool rather than a raw form.
  • Zero external dependencies — no API key modal, no network calls, no backend. Everything (including the markdown renderer) is hand-written so the file works fully offline, double-clicked from disk.

Future extensions

  • Tools: connect a real LLM API (Claude, etc.) behind a backend proxy for genuinely generative, non-templated output; add a "trend lookup" tool to ground hooks in current platform trends; add a brand-voice analyzer that ingests past posts to extract vocabulary and rhythm patterns automatically.
  • Memory: persist the user's niche, audience, and tone in localStorage across sessions so the brief pre-fills and the strategist references what's worked before ("Your last post about pricing got strong saves — want a follow-up?").
  • Multi-step workflows: chain this into a content calendar mode — generate a week of interlinked posts building one narrative arc, or a repurposing workflow that turns one long-form idea into platform-native variants automatically.
