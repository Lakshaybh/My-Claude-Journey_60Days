# ENVIRONMENT.md — QueryMind

## 1. Tools & Their Purpose

| Tool | Why it's needed |
|---|---|
| Python 3.11+ | The language the entire backend is written in. Already installed (3.13.7). |
| `venv` (built into Python) | Creates an isolated set of packages just for this project so nothing conflicts with other Python projects on your machine. |
| `pip` | Installs Python packages from `requirements.txt`. |
| Uvicorn | The web server that actually runs the FastAPI app and listens for browser requests. |
| Git | Version control — tracks every change, already connected to GitHub. |

No IDE-specific extensions are strictly required, but if using VS Code, the **Python extension** (by Microsoft) is recommended — it gives you syntax highlighting, linting, and lets you pick the `venv` interpreter from the bottom-right corner of VS Code so autocomplete works correctly.

## 2. Python Package Dependencies (`requirements.txt`)

| Package | Purpose |
|---|---|
| `fastapi` | The web framework — defines routes, validates requests automatically via Pydantic, and gives us a free interactive API tester at `/docs`. |
| `uvicorn[standard]` | Runs the FastAPI app as an actual running web server. |
| `anthropic` | Official Python SDK for calling the Claude API — used starting Day 4. |
| `python-dotenv` | Loads variables from `.env` into the app so secrets never get hard-coded into source code. |

## 3. Environment Variables

| Variable | Where it's used | Required today? |
|---|---|---|
| `ANTHROPIC_API_KEY` | `app/core/config.py` — will be passed to the Anthropic SDK starting Day 4 | No — only required starting Day 4 |

**File:** `.env` (in `Day 52/querymind/`) — created by copying `.env.example` and filling in the real key. This file is git-ignored (see `.gitignore`) and must never be committed.

## 4. Configuration Files

| File | Purpose |
|---|---|
| `.env.example` | Documents which environment variables are needed, without real secret values — safe to commit. |
| `.gitignore` | Excludes `venv/`, `__pycache__/`, `*.pyc`, `*.log`, and `.env` from version control. |
| `requirements.txt` | Pinned list of Python dependencies, used both locally and by Render at deploy time (Day 9). |
